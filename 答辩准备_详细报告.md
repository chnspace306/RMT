## 毕业设计答辩详细报告

## 第一章：项目背景与研究意义

### 1.1 问题背景

在量化金融、生物信息学、神经科学、环境监测等现代数据分析领域，研究者面对的数据矩阵往往具有**高维度**（特征数 p 很大）和**有限样本**（观测数 n 有限）的特点。传统统计方法（如直接观察相关系数矩阵）在 p/n 比例较高时会产生严重偏差——即使数据完全是随机噪音，相关矩阵也会表现出虚假的结构。

**核心矛盾**：如何从一个受到有限样本噪声污染的高维数据矩阵中，可靠地区分出真实信号和随机噪音？

### 1.2 随机矩阵理论的解决思路

随机矩阵理论（Random Matrix Theory, RMT）提供了精确的数学工具来解决这一问题。RMT 的核心洞见是：

- 对于一个**纯随机**的高维矩阵，其特征值的统计分布在极限下收敛于确定的解析形式（如 Wigner 半圆律、Marchenko-Pastur 律）
- 任何**偏离**这一理论预测的特征值，都代表着数据中存在的非随机结构——即真实信号

这使得 RMT 成为高维数据**信噪分离**的理想理论框架。

### 1.3 项目的实际价值

本平台将 RMT 的理论知识转化为**可视化、交互式、可解释**的分析工具，使研究者能够：

1. 上传真实数据集，自动计算特征值谱并对比理论预测
2. 识别并解剖异常特征值对应的变量成分（特征向量分解）
3. 使用 RMT 清洗方法过滤噪音相关矩阵
4. 沿时间轴监测系统性风险的动态演化
5. 借助 AI 获得专业的统计洞察报告

## 第二章：核心数学理论基础

### 2.1 随机矩阵系综

**高斯正交系综 (GOE)**

对于一个 N×N 的实对称随机矩阵 H：
- 对角线元素：Hᵢᵢ ~ N(0, 2σ²)
- 非对角线元素：Hᵢⱼ ~ N(0, σ²)，且 Hᵢⱼ = Hⱼᵢ

在项目中的实现（`rmt_core.py` 第 4-17 行）：
```python
A = np.random.normal(loc=0.0, scale=scale, size=(n, n))
H = (A + A.T) / np.sqrt(2.0)
eigenvalues = np.linalg.eigvalsh(H)
```

**样本协方差矩阵系综**

对于 n×p 的矩形随机矩阵 X（每行一个样本，每列一个特征），样本协方差矩阵为：
C = (1/n) · XᵀX

在项目中的实现（`rmt_core.py` 第 29-37 行）：
```python
X = np.random.normal(loc=0.0, scale=scale, size=(n, p))
C = (1.0 / n) * np.dot(X.T, X)
eigenvalues = np.linalg.eigvalsh(C)
```

### 2.2 Wigner 半圆律

**定理**：对于 GOE 矩阵，当 N → ∞ 时，特征值 λ 的经验谱密度收敛于半圆分布：

ρ(λ) = (2 / πR²) · √(R² − λ²)，  |λ| ≤ R

其中 R = 2σ√N 为半圆半径。

**物理意义**：半圆律是最基础的 RMT 普适性结果，它说明随机对称矩阵的特征值不会均匀分布，而是聚集成半圆形状——边缘处密度最高，中心反而较低。这一性质与经典的中心极限定理完全不同的统计行为。

### 2.3 Marchenko-Pastur (MP) 律

**定理**：对于 n×p 随机矩阵 X（元素独立同分布，均值为 0，方差为 σ²），当 n, p → ∞ 且 q = p/n ∈ (0, ∞) 时，样本协方差矩阵 C = (1/n)XᵀX 的特征值经验谱密度收敛于 MP 分布：

ρ(λ) = √[(λ₊ − λ)(λ − λ₋)] / (2πqσ²λ)，  λ ∈ [λ₋, λ₊]

其中：
- λ₊ = σ²(1 + √q)²  — 理论上界
- λ₋ = σ²(1 − √q)²  — 理论下界（若 q > 1，存在零特征值的原子质量）

**这是本项目最核心的理论工具**。MP 律的直接推论：
- **所有落在 [λ₋, λ₊] 内的特征值都是噪音**：它们与纯随机矩阵的行为无法区分
- **越过 λ₊ 的特征值代表真实信号**：它们在统计上不可能由纯随机产生，必然对应数据中存在的真实结构
- **信号强度**：特征值越大，对应的主成分解释的方差越多，代表的"信号"越强

### 2.4 特征向量与成分解剖

特征值告诉我们"有多少信号"，但特征向量告诉我们"信号是什么"。

对于协方差矩阵 C 的每个特征值 λₖ 和对应的特征向量 vₖ（单位长度）：

C · vₖ = λₖ · vₖ

特征向量 vₖ 的每个分量 vₖⱼ 代表第 j 个原始变量在该主成分上的**载荷（Loading）**：
- **正载荷**（vₖⱼ > 0）：该变量与主成分同向共动
- **负载荷**（vₖⱼ < 0）：该变量与主成分反向运动（跷跷板关系）
- **载荷绝对值大**：该变量是此主成分的主导力量

项目中对每个异常特征值提取 Top-5 高权重变量，为 AI 分析和用户解读提供精确的变量级证据。

### 2.5 逆参与率 (IPR)

IPRₖ = Σⱼ |vₖⱼ|⁴

- **IPR ≈ 1/N**（N 为变量数）：延展态，能量均匀分布在所有变量上 → 噪音特征值
- **IPR → 1**：局部化态，能量集中在少数变量上 → 异常信号特征值

IPR 提供了另一种独立于特征值大小的信号检测手段。

### 2.6 RMT 相关矩阵清洗

核心思想：将样本协方差矩阵的特征值分解后，仅保留信号特征值（λ > λ₊），将噪音特征值替换为其均值，重构"清洗后"的相关矩阵：

C_clean = Σ_{λₖ > λ₊} λₖ · vₖ · vₖᵀ  +  λ̄_bulk · Σ_{λⱼ ≤ λ₊} vⱼ · vⱼᵀ

这种方法能有效消除由有限样本引起的虚假相关性。

### 4.2 API 端点详解

#### 4.2.1 POST /api/rmt/wigner

**功能**：生成 GOE 随机矩阵并返回 Wigner 半圆律分析结果

**请求体**：
```json
{
  "n": 1000,       // 矩阵维度
  "scale": 1.0     // 元素标准差缩放因子
}
```

**响应体**：
```json
{
  "eigenvalues": [2.01, -1.98, ...],  // N 个特征值
  "theoretical_curve": {
    "x": [-2.2, -2.19, ...],           // 理论曲线的 x 坐标 (500 个点)
    "y": [0.01, 0.015, ...]            // 对应的半圆律密度值
  }
}
```

**算法流程**：
1. 生成 N(0, scale²) 分布的 N×N 矩阵 A
2. 对称化 H = (A + Aᵀ) / √2（保持非对角线方差为 scale²）
3. `np.linalg.eigvalsh(H)` 计算特征值
4. 计算理论半径 R = 2 · scale · √N
5. 在 [-1.1R, 1.1R] 上生成 500 个等距点，计算半圆律 PDF

#### 4.2.2 POST /api/rmt/mp

**功能**：生成矩形随机矩阵并返回 MP 律分析结果

**请求体**：
```json
{
  "n": 1000,       // 样本数
  "p": 350,        // 特征数
  "scale": 1.0     // 标准差缩放因子
}
```

**响应体**：
```json
{
  "q": 0.35,                    // p/n 维度比
  "lambda_plus": 2.84,          // MP 理论上界
  "lambda_minus": 0.12,         // MP 理论下界
  "eigenvalues": [...],         // p 个特征值
  "theoretical_curve": {...}
}
```

**算法流程**：
1. 计算 q = p / n
2. 生成 N(0, scale²) 分布的 n×p 矩阵 X
3. C = (1/n) · XᵀX
4. `np.linalg.eigvalsh(C)` 计算 p 个特征值
5. λ₊ = scale²(1+√q)²，λ₋ = scale²(1-√q)²
6. 在 [max(0.01, λ₋-0.5), λ₊+0.5] 上生成理论 MP 曲线

#### 4.2.3 POST /api/rmt/upload

**功能**：上传 CSV 文件并进行完整的 RMT 分析（这是最复杂、功能最丰富的端点）

**请求**：multipart/form-data
- `file`: CSV 文件
- `scale`: 缩放因子
- `fill_strategy`: "zero" | "mean" | "drop"
- `standardize`: "true" | "false"

**算法流程（共 6 个阶段）**：

**阶段 1：CSV 解析与表头检测**（第 80-96 行）
```
读取原始文本 → 用 csv 模块解析首行 →
试探每列是否为数字 → 若非数字 → 判定为表头 →
提取列名供后续成分分析使用
```

**阶段 2：缺失值处理**（第 111-121 行）
```
fill_strategy = "zero"  → np.nan_to_num(mat, nan=0.0)
fill_strategy = "mean"  → 每列用列均值填充 NaN
fill_strategy = "drop"  → 删除含 NaN 的行
```

**阶段 3：标准化与协方差计算**（第 129-137 行）
```
若 standardize=true → 每列 (x - μⱼ) / σⱼ（σⱼ=0 的列设为 1 防除零）
centered = X - column_means
C = (1/n) · centeredᵀ · centered
eigenvalues, eigenvectors = np.linalg.eigh(C)  ← 关键：同时获取特征向量
```

**阶段 4：异常特征向量成分提取**（第 152-187 行）
```
将特征值从大到小排序 →
对每个 > λ₊ 的特征值：
  取其对应的特征向量 →
  计算每个分量的绝对值 →
  按绝对值排序取 Top-5 →
  匹配列名（若有表头）或生成 Column_N →
  记录权重、正负号、列索引
```
最多前 10 个异常特征值的分解结果，每个含 Top-5 成分。

**阶段 5：IPR 计算**（第 190-193 行）
```python
ipr = np.sum(eigenvectors**4, axis=0)
```

**阶段 6：RMT 清洗热力图**（第 195-243 行）
```
识别信号特征值（λ > λ₊）→
计算噪音特征值的均值 λ̄_bulk →
重构清洗矩阵：
  C_clean = Σ(λₖ · vₖ · vₖᵀ) 仅对信号值
          + λ̄_bulk · Σ(vⱼ · vⱼᵀ) 对噪音值
对角线归一化 → matplotlib coolwarm 配色 →
并排展示原始 vs 清洗矩阵 → 输出 base64 PNG
```

**核心响应字段**：
```json
{
  "q": 0.52,
  "lambda_plus": 3.15, "lambda_minus": 0.08,
  "eigenvalues": [...],
  "theoretical_curve": {...},
  "sparsity": 0.12,
  "n": 500, "p": 260,
  "outlier_eigenvectors": [
    {
      "eigenvalue": 45.23,
      "rank": 1,
      "top_components": [
        {"column_index": 3, "column_name": "贵州茅台", "weight": 0.82, "abs_weight": 0.82},
        {"column_index": 7, "column_name": "五粮液",   "weight": 0.76, "abs_weight": 0.76},
        ...
      ],
      "vector": [0.03, -0.01, 0.02, ...]
    }
  ],
  "column_names": ["日期", "贵州茅台", "五粮液", ...],
  "ipr": [0.0012, 0.0015, ...],
  "cleaned_heatmap_base64": "data:image/png;base64,..."
}
```

#### 4.2.4 POST /api/rmt/rolling

**功能**：沿时间轴的滚动窗口最大特征值分析

**算法**：
```
读取 CSV → 检测时间列 → 窗口大小 window_size=60 →
步长 step_size=5 → 对每个窗口：
  [start : start+window_size] 的矩阵 →
  标准化（可选）→ 计算协方差矩阵 →
  eigh 分解 → 取最大特征值 λ₁ →
  记录对应时间点
```

**输出**：`{times: [...], lambda_1: [...]}`，前端用 ECharts 折线图渲染。

**应用意义**：λ₁ 的时序变化反映系统性风险的动态演化。在金融危机期间，股票收益率相关矩阵的 λ₁ 会急剧飙升。

#### 4.2.5 POST /api/rmt/heatmap_rebuild

**功能**：独立的热力图重建，支持 `top_k` 参数仅保留前 N 个信号

**关键参数**：`top_k` 控制保留的信号特征值数量，-1 = 保留全部异常值

这使得用户可以交互式地探索"只保留最强的前 K 个信号时，清洗后的相关矩阵长什么样"，逐层剥离信号进行层析分析。

#### 4.2.6 POST /api/rmt/analyze

**功能**：AI 驱动的统计洞察生成

**Prompt 设计（第 288-307 行）**：
```
【核心数据指标】
- 维度、q 值、稀疏度
- MP 边界、越界特征值数量
- 头部主成分特征值
【特征向量成分解剖】(新增)
- 每个异常特征值对应的高权重变量名、权重符号
【任务要求】
1. 稀疏度和 q 值的宏观影响
2. 基于列名精确判断异常特征值的物理/业务含义
3. 信噪比结论
```

**特色**：prompt 中明确要求 AI 分析特征向量权重的正负号，判断"跷跷板"对抗关系。

**传输方式**：SSE (Server-Sent Events) 流式响应，前端实时渲染。

### 5.3 RmtChart.vue 谱图组件

**核心算法**：自适应分箱直方图

```
createHistogram(data, binsCount):
  1. 计算 min, max
  2. 若存在极端异常值（max > 2×λ₊），将 coreMax 裁剪到 1.2×λ₊
  3. binWidth = (coreMax - min) / binsCount
  4. 遍历所有特征值，计算每个落入的 bin
  5. 根据 normType 输出 density 或 count
```

**可视化元素**：
- 蓝色渐变柱 = 噪音区域（bulk）
- 橙红渐变柱 = 异常信号（超出理论边界）
- 绿色曲线 = 理论密度曲线
- 红色虚线 = MP/Wigner 理论边界
- 内建 dataZoom 滑块 + 鼠标滚轮缩放

**图表 → 面板联动**：
```
chartInstance.getZr().on('click') →
判断点击坐标是否落在异常区域 →
计算最近的异常特征值 →
emit('outlier-click', anomalyIndex)
```

### 7.1 特征向量成分解剖

传统 RMT 分析仅告诉你"有多少个信号特征值"。本项目进一步回答了**"每个信号代表什么"**：

## 第九章：答辩可能问题与回答思路

### Q1: 为什么选择 RMT 而不是传统 PCA？

传统 PCA 无法区分"真实的因子结构"和"由有限样本引起的随机波动"。在 p/n 较高时，即使数据完全是随机噪音，PCA 也会产生看似显著的主成分。RMT 通过 MP 律提供了精确的统计检验框架——只有越过 λ₊ 的特征值才被认为是统计显著的。

### Q2: MP 律的适用条件是什么？你的数据满足吗？

MP 律要求矩阵元素独立同分布、均值 0、方差有限。实践中，真实数据很少完全满足 i.i.d.，但 MP 律具有普适性——即使元素分布偏离正态、存在弱相关性，bulk 区域的谱分布仍收敛于 MP 律。项目中通过标准化预处理使数据近似满足条件。

### Q3: q = p/n 的意义是什么？

q 值决定了 MP 分布的展宽。q 越大（特征数相对样本数越多），MP 边界越宽，噪声区域越大，越难检测信号。这解释了"维度诅咒"：当特征数接近或超过样本数时，噪音会淹没信号。

### Q4: 特征向量的权重正负号代表什么？

正权重意味着该变量与主成分正相关（同涨同跌），负权重意味着负相关（此消彼长）。同一特征向量中正负混合的变量形成"跷跷板"关系——例如在金融中，防御性板块（正权重）与周期性板块（负权重）的对立。

### Q5: RMT 清洗矩阵相比原始矩阵丢失了什么信息？

RMT 清洗仅过滤了统计上无法与随机噪音区分的特征值（即 bulk 区域）。这部分信息在统计意义上确实是噪音——它们由有限样本引起的伪相关性构成。清洗后的矩阵保留了所有统计显著的结构，实际上更准确地反映了变量间的真实关系。

### Q6: 为什么使用 eigh 而不是直接的相关矩阵特征分解？

`np.linalg.eigh` 专门为对称/厄米矩阵优化，利用矩阵的对称性使用更高效的算法。更重要的是，它保证返回的特征向量是正交的，这对于后续的成分分析和热力图重建至关重要。

### Q7: AI 分析的可靠性如何保证？

AI 分析作为辅助解释工具而非权威结论。项目将精确的数值证据（特征值、权重、列名）注入 prompt，AI 基于这些硬数据进行推断。最终的解释责任在研究者——AI 提供思路，数值证据提供支撑。
